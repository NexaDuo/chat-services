<objective>
Establish the reproducible, cost-effective infrastructure baseline on Google Cloud Platform using Terraform.

Purpose: Provide a stable compute, network, and storage foundation for the NexaDuo multi-tenant chat platform.
Output: Fully provisioned GCP environment with Cloudflare DNS wiring.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Terraform Project Initialization</name>
  <action>
    Initialize the Terraform workspace and define the GCP provider and remote backend (if applicable).
    1. Navigate to `infrastructure/terraform/envs/production/`.
    2. Run `terraform init`.
  </action>
  <verify>
    <automated>terraform version</automated>
  </verify>
  <done>Terraform is initialized and provider plugins are downloaded.</done>
</task>

<task type="auto">
  <name>Task 2: Provision Core Compute and Storage</name>
  <action>
    Apply the Terraform configuration to create the VM instance and persistent disk.
    1. Define `e2-standard-4` machine type in `variables.tf`.
    2. Define `pd-balanced` 50GB disk in `main.tf`.
    3. Run `terraform apply -auto-approve`.
  </action>
  <verify>
    <automated>gcloud compute instances describe nexaduo-app --format="get(machineType)"</automated>
  </verify>
  <done>GCP Compute instance and persistent storage are provisioned.</done>
</task>

<task type="auto">
  <name>Task 3: Configure Security and Networking</name>
  <action>
    Configure VPC and Firewall rules to enforce the "Portless Origin" model.
    1. Create custom VPC.
    2. Restrict SSH ingress to Google Identity-Aware Proxy (IAP) IP range (35.235.240.0/20).
    3. Ensure no public 80/443 ports are open on the VM.
  </action>
  <verify>
    <automated>gcloud compute firewall-rules list --filter="network:nexaduo-vpc" --format="table(name,sourceRanges,allowed)"</automated>
  </verify>
  <done>VPC and IAP-only firewall rules are applied.</done>
</task>

<task type="auto">
  <name>Task 4: Cloudflare DNS Integration</name>
  <action>
    Initialize Cloudflare DNS records for core services.
    1. Define A/CNAME records for `chat.nexaduo.com` and `dify.nexaduo.com`.
    2. Apply via Terraform.
  </action>
  <verify>
    <automated>dig +short chat.nexaduo.com</automated>
  </verify>
  <done>Cloudflare DNS records point to the provisioned infrastructure.</done>
</task>

</tasks>

<success_criteria>
- `terraform apply` completes successfully.
- VM instance is running with `e2-standard-4` specs.
- Firewall blocks all public ingress except via IAP for SSH.
- DNS records are correctly created in Cloudflare.
</success_criteria>
