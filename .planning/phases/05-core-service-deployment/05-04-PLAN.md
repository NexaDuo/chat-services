<objective>
Deploy the largest stack — Middleware bridge, Evolution API, observability (Loki, Promtail, Grafana, NEW Prometheus), and self-healing agent — via Terraform-managed Coolify. Resolves three blocking issues identified in research: (1) Prometheus container is missing from any compose file (DEPLOY-04 gap), (2) middleware and self-healing-agent use `build:` directives Coolify cannot resolve relative to its working dir (Pitfalls 5/6), (3) Grafana's Postgres datasource requires `POSTGRES_USER`/`POSTGRES_PASSWORD` env vars at runtime (Pitfall 3).

Output: Updated `docker-compose.nexaduo.yml` with prometheus service + image tags (no build context); Terraform `coolify_service.nexaduo` + envs + verify probe wired into `main.tf`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local dev / CI runner -> Coolify API | Bearer-token requests to add the nexaduo service + env vars |
| Local dev / CI runner -> GCP VM (SSH) | `null_resource.verify_nexaduo` opens SSH for the multi-endpoint probe |
| WhatsApp -> Evolution API (port 8080) | Evolution receives messages from Meta; protected by EVOLUTION_AUTHENTICATION_API_KEY |
| Chatwoot webhook -> Middleware (port 4000) | Middleware /webhook trusts Chatwoot-origin requests inside the Docker network |
| Dify HTTP Tool -> Middleware /tools/handoff | Authenticated via HANDOFF_SHARED_SECRET Bearer token |
| Promtail -> /var/run/docker.sock (read-only) | Promtail tails container stdout via the Docker socket |
| Operator -> Grafana (port 3002) | Admin UI behind Cloudflare Tunnel; basic auth via GRAFANA_ADMIN_PASSWORD |
