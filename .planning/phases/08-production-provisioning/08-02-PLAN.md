    1. Obtain the Chatwoot Platform Token from the production VM:
       `gcloud compute ssh nexaduo-chat-services --tunnel-through-iap --command "sudo docker exec nexaduo-chat-rails bundle exec rails runner 'p PlatformApp.first_or_create!(name: \"provisioning\").access_token.token'"`

    4. Follow the manual steps in `provisioning/README.md` to:
       - Create an app in Dify Console.
       - Update the `TENANT_MAP` in the production environment (via Coolify UI or `gcloud ssh`).
       - Restart the middleware service.
  </action>

    <automated>grep -q "NexaDuo Main" provisioning/tenants.json</automated>
  </verify>
  <done>First production tenant created in Chatwoot and registered in local tracking.</done>
