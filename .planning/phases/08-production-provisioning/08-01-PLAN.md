  <files>infrastructure/terraform/envs/production/providers.tf</files>
  <action>
    Set up the Coolify API token in GCP Secret Manager.
    1. Log in to the new Coolify instance at `http://&lt;VM_IP&gt;:8000`.
    2. Go to "Settings" -> "API Tokens".
    3. Create a new token named "terraform-production".

       `echo -n "YOUR_TOKEN" | gcloud secrets versions add coolify_api_token --data-file=-`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terraform → GCP | Terraform uses Service Account credentials to provision infrastructure. |
| Terraform → Coolify | Terraform uses API Token to manage services. |
| Edge → Origin | Cloudflare Tunnel restricts traffic to authenticated tunnel requests only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | Secrets | Mitigate | Secrets are never stored in state or locally; fetched dynamically from GCP Secret Manager (per VAULT-01). |
| T-08-02 | Spoofing | Coolify API | Mitigate | Coolify API access restricted to localhost/internal only; Terraform uses unique API token. |
| T-08-03 | Information Disclosure | Terraform State | Mitigate | State stored in a restricted GCS bucket with encryption (per Phase 1 backend config). |
</threat_model>
