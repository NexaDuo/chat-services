<objective>
Centralize and secure all project secrets using GCP Secret Manager as the Single Source of Truth (SSOT).

Purpose: Remove sensitive data from source control (local .tfvars) and provide a consistent secret management strategy across development and production environments.
Output: Integrated secret management workflow using GCP Secret Manager.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Project Setup & Migration</name>
  <action>
    Initialize Secret Manager and migrate existing local secrets.
    1. Enable Secret Manager API in the GCP project.
    2. Create and run `scripts/migrate-to-gcp-secrets.sh` to push local `.tfvars` to GCP.
  </action>
  <verify>
    <automated>gcloud secrets list</automated>
  </verify>
  <done>All sensitive project secrets are stored in GCP Secret Manager.</done>
</task>

<task type="auto">
  <name>Task 2: Terraform Integration</name>
  <action>
    Update Terraform to fetch secrets from GCP and inject into Coolify.
    1. Update `infrastructure/terraform/envs/production/main.tf` to use `google_secret_manager_secret_version` data sources.
    2. Map fetched secrets to `coolify_service_envs` resources.
    3. Remove sensitive defaults from `variables.tf`.
  </action>
  <verify>
    <automated>terraform plan (checking for sensitive data masking and source=GCP Secret Manager)</automated>
  </verify>
  <done>Terraform fetches all sensitive values from Secret Manager at runtime.</done>
</task>

<task type="auto">
  <name>Task 3: Local Environment Sync</name>
  <action>
    Create a local sync workflow to pull project secrets for development.
    1. Create `scripts/sync-secrets-gcp.sh` to pull GCP secrets to a local `.env`.
    2. Ensure `.env` and `*.tfvars` are strictly ignored in `.gitignore`.
  </action>
  <verify>
    <automated>ls -a .env && grep -v "^#" .env</automated>
  </verify>
  <done>Local environment can securely sync secrets from GCP.</done>
</task>

</tasks>

<success_criteria>
- GCP Secret Manager is enabled and populated.
- Terraform configuration uses Secret Manager data sources.
- No sensitive data remains in `terraform.tfvars` or `variables.tf`.
- Local `.env` is successfully synchronized from GCP.
</success_criteria>
