## Infrastructure (INFRA)

- [x] **INFRA-01**: Provision GCP Compute Instance (e2-standard-4) using Terraform.
- [x] **INFRA-02**: Configure GCP Networking (VPC, Firewall for SSH/HTTP/S, Static IP) via Terraform.
- [x] **INFRA-03**: Provision Persistent SSD Storage (50-100GB) via Terraform.
- [x] **INFRA-04**: Install and initialize Coolify v4 on the provisioned VM.
- [x] **INFRA-05**: Implement automated backup rotation to GCS (Google Cloud Storage).
- [ ] **INFRA-06**: Automate Coolify v4 application and service configuration via Terraform/API.
  > **Deferred:** Manual Coolify UI used throughout v1.0. API automation out of scope for MVP.

## Routing & Edge (ROUTE)

- [x] **ROUTE-01**: Configure Cloudflare DNS for unified subdomains (`chat.nexaduo.com`, `dify.nexaduo.com`).
- [x] **ROUTE-02**: Implement Cloudflare Worker for path-based tenant identification (`/tenant/`).
- [x] **ROUTE-03**: Inject `X-Tenant-ID` header at the edge for backend consumption.
- [x] **ROUTE-04**: Secure origin by restricting access to Cloudflare IPs or using Cloudflare Tunnels.

## Application Deployment (DEPLOY)

- [x] **DEPLOY-01**: Orchestrate Chatwoot, Dify, and Evolution API using Coolify/Docker Compose.
- [x] **DEPLOY-02**: Configure Chatwoot and Dify for path-based multi-tenancy via Cloudflare Workers.
- [x] **DEPLOY-03**: Deploy the Middleware bridge to handle Chatwoot-Dify communication.
- [x] **DEPLOY-04**: Deploy Observability stack (Prometheus/Grafana) within the same environment.

## Multi-tenancy & Provisioning (PROV)

- [x] **PROV-01**: Define a standardized tenant configuration schema.
  > Delivered in Phase 4 (04-01 Provisioning CLI — Zod schema + tenants.json structure).
- [x] **PROV-02**: Automate DNS record creation for new tenants via Terraform.
  > Delivered in Phase 4 (04-02 — Terraform for_each over tenants.json).
- [x] **PROV-03**: Automate Cloudflare Worker routing table updates for new tenants.
  > Delivered in Phase 4 (04-02/03 — Terraform automation + E2E verification).
